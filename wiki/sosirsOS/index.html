Программирование в sosirsOS

В sosirsOS присутствуют два языка программирования для создания программ для sosirsOS: InLangScript (главный, основной) и Felan (Для взаимодействия с RAM и VRAM (эмуляторов)), также в планах Catton

Для выполнения когда надо создать файл, легче всего через sosirsOS: нажимаешь на иконку, вводишь пароль, нажимаешь кнопку "create", вводишь имя файла (например my code), вводишь id типа файла (22 — InLangScript, 23 — Felan), вводишь код (содержание)

Внимание: Необязательно сразу же нажимать Create как только открывается интерфейс (всегда открывается sys/sosirsOS:B:), вы можете в другой папке, или создать свою папку: нажать Create, ввести название папки, ввести 14 (id типа папок) и папка будет создана, чтобы она появилась надо перезайти в sosirsOS Files, либо зайти в другую папку, а потом нажать "все", выбрать sys, выбрать sosirsOS, выбрать B:, и папка появится (на одной странице до 6 файлов, чтобы посмотреть все файлы которые невместились надо нажать ⬆️, обратно ⬇️)

Дальше для выполнения кода надо выполнить код: выбираем файл InLangScript или Felan, вводим 1, откроется терминал для запуска, вводим 3 и код выполниться

Создание дистрибутивов

Для создания дистрибутивов sosirsOS надо разбираться в: InLangScript, Felan, файловой системе sosirsOS, знать хорошо Scratch и разбираться в структуре sosirsOS

Файловая система sosirsOS — штука которая нужна для хранения данных как файлов, обмена файлами и другого. Для взаимодействия через интерфейс существует одноимённое приложение sosirsOS Files

sosirsOS Files для входа в интерфейс требует ввод пароля от системы

Файлы хранятся в элементах списка "file data" и "files names", всё устроено так: файлы которые хранят в себе другие файлы (например папки и архивы) имеют 3 элемента: 1-ый это имя, 2-ой это тип (id типа в sosirsOS Files) и путь, например sys/sosirsOS:B:apps/ или root/home/, а файлы которые содержат в себе текст (например текстовые файлы, файлы InLangScript и т.д.) имеют 4 элемента, первые три это такие же как и в тех которые содержат другие файлы, а четвертый это содержание

Заменять sosirsOS везде на название вашего дистрибутива — бессмысленно, слишком много таких файлов, и не удаляйте их, пусть они останутся как название системного компонента, лучше вместо переименования и затраты большого количества времени, лучше создайте отдельную папку и назовите так же как и ваш дистрибутив называется (например, myDistroSosirsOS по пути sys/sosirsOS:A:), исключение: содержимое файла os.txt или подобных

Способ добавлять приложения из приложения для скачивания на рабочий стол — очень странный, лучше все делать как InLangScript коды, как делается сейчас

Рекомендуется при каждой новой версии sosirsOS (или через каждые сколько-то версий) делать новую версию вашего дистрибутива чтобы не уходить назад из тренда и иметь самые мощные технологии. Минус: Свои функции надо писать с нуля

Создание систем похожих на sosirsOS

Создание похожих систем — интересный процесс.

Для него лучше всего понимать устройство sosirsOS, иначе как поймёшь как делать похожую систему.

Надо изучить код sosirsOS, понять что делает что, например, понять что в терминале отвечает за выполнение InLangScript

Лучше всего конечно делать дистрибутив sosirsOS так как там уже есть готовая реализация многих функций, но можно сделать и дистрибутивы систем похожих на sosirsOS (и основанных по идее sosirsOS).

Но, если вы сделаете дистрибутив sosirsOS, а потом перепишите на условный C или Java тогда вы уже будете является создателем похожей на sosirsOS системы, так как дистрибутив sosirsOS — это в первую очередь система основанная на коде sosirsOS. А если переписать дистрибутив sosirsOS на другой язык программирования это уже не будет система основанная на исходном коде, а написанная с нуля система, хоть и полностью выглядит как дистрибутив sosirsOS который вы сделали.

Возможно вы захотите назвать черную квадратную маленькую кнопку с округленными краями которая находится в правом меню (например как в Ubuntu, но в другой стороне) и меню которое она открывает "Пуск", но это меню называется в экосистеме sosirsOS "меню"

Документация по парсерам

Создание парсеров — важный процесс в разработке операционной системы

Парсер нужен чтобы разбить строку на пару элементов, он нужен чтобы например найти команду в "print/hello" где парсер разделяет строку символом '/', и эта строка конвертируется в список, код возьмет команду из элемента 1 и получит команду 'print', команда 'print' обратится к элементу 2 и получит 'hello', и код выведет 'hello'

Если хотите добавить язык программирования, видеоплеер или формат фотографий — вам нужен парсер

Для создания парсера надо выполнить следущие шаги:

1. Создайте новую функцию в коде и назовите её "decode", "декодировать" или ещё как-то, поставьте шлажок "Запустить без обновления экрана", это нужно чтоб код выполнился как можно быстрее

2. Создайте переменные только для спрайта: i и i2

3. Задайте переменной i значение 1

4. Сделайте цикл "пока не" с boolean которое значит что переменная i больше длины переменной значение которое код будет парсить (напрмер переменная code)

4.1. В цикле задайте переменной i2 значение "" (ничего)

4.2. Добльше добавьте в цикл ещё один цикл "пока не" с boolean котрое значит что буква с номером который равен значению переменной i в условной переменной code равна i

4.2.1. А в этом цикле задайте переменной i2 значение которое равно значению i2 обьедененного с значением буквы в условной переменной code с номером равным переменной i

4.2.2. Добавьте блок для изменения переменной i на 1

4.3. Дальше в основном цикле добавьте к условному списку decode элемент со значением переменной i2

4.4. Тоже измените переменную i на один

5. Уже не в циклах добавьте к условному списку элемент со значением переменной i2

6. Скрипт парсера завершён

Парсеры выполняют важную роль в sosirsOS, например содержание текстового файла в sosirsOS — минимум и максимум одна строка, поэтому например для запуска много строчного кода надо делать кучу строк в одну, а для запуска — обратно, а как раз чтобы обратно это сделать нужен парсер
